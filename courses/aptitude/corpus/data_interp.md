# Data Interpretation

## Reading Tables
Tables show values at the intersection of rows and columns.
Steps: (1) identify what each row/column represents, (2) read the question carefully, (3) extract only the relevant cells.

Example: A table shows sales (in thousands) by quarter and region. "Total sales in Q2" = sum of all Q2 cells across regions.

## Bar Charts
Each bar's height or length represents a value. Compare bars visually but always read exact values from the axis.

Percentage share of one bar = (bar value / total of all bars) × 100.

## Pie Charts
A pie chart represents 100% (360°). Each sector's angle = (value / total) × 360.
To find value from angle: value = (angle / 360) × total.

Example: A sector of 90° in a chart with total = 1200 represents (90/360)×1200 = 300.

## Line Graphs
Show trends over time. Key tasks: find maximum/minimum, % change between two points, rate of change.

% change from year A to year B = (Value_B − Value_A) / Value_A × 100.

## Common DI Mistakes to Avoid
- Confusing the y-axis scale (thousands vs. lakhs vs. crores).
- Missing that values are approximate — read exact numbers from labels.
- Calculating percentage of total using wrong denominator.

## Approach for DI Sets
1. Spend 30 seconds understanding the chart before reading questions.
2. Identify the unit (%, Rs, persons, etc.).
3. For multi-chart DI sets, note which chart answers which question type.
